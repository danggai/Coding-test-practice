## 체감 난이도

★★★★ (4점/5점)



정확한 문제는 네이버 지침 상 공개할 수 없고, 문제에 대한 자세한 언급 또한 하지 않음.

---

네이버 신입 공채 코딩테스트.



문제의 난이도를 떠나서 개인적인 불만사항이 있었다.

가장 큰 불만으로는 IDE 사용과 웹 검색을 제한한 것.



IDE 사용 제한이야 충분한 환경을 제공해준다면 불만이 없었겠지만, 프로그래머스에서 제공하는 IDE는 아직 매우 불편하다.

심지어 프로그래머스 내 복사+붙여넣기도 불가능하여 반복적으로 사용하는 변수를 일일히 다시 입력해야하는 부분이 굉장히 짜증났다.





1번 문제는 복잡하지 않았다.

문제에서 주어진 순서대로만 반복문을 짜도 테스트케이스에 한해서는 해결할 수 있었다.

자세한 채점 내용은 공개해주지 않아서 답답했다.



2번 문제는 다중 큐에대한 문제였으나, 역시 복잡하지 않았다.

다중 큐에 대한 구현에 추가로, 모든 다중 큐가 공유하는 영역에 대한 구현이 관건이였다.



3번 문제는 적당히 어려워 보였으나, 남은 시간이 부족했고 위에서 언급한 사항으로 인해 심적으로 피곤해 구현하지 않았다.

30분 정도가 남아있었는데, 1시간 정도면 할 만 했을지도..?

주어진 배열을 주어진 방법에 따라 스왑하는 함수를 구현하는 문제였다.



4번까지 있었는데 4번은 구경도 못했다ㅠ
